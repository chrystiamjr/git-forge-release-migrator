# Pull Request Template

Use this structure for every PR opened against `main`.

```markdown
## Summary

<!-- One to three sentences: what this PR does and why it exists. -->

## Context

<!-- What was the state before this PR?
     Why is this change needed now?
     Link related issues, tickets, or prior PRs if relevant. -->

## What Changed

<!-- Bullet list grouped by concern.
     Focus on behavior and structure, not file names. -->

- **<Area or component>:** <description of change and rationale>

## Why It Matters

<!-- What problem does this solve?
     What risk does it remove?
     What capability does it add? -->

## Expected Results

<!-- What should reviewers verify? -->

```bash
yarn test:dart
```

## Additional Notes

<!-- Optional follow-up work, risky areas, or deployment notes. -->
```
